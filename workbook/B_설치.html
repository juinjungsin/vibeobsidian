모듈 B. 설치

Node.js → 코덱스 CLI → 옵시디언 → Claudian 플러그인 순서로 설치한다 · ⏱ 90~120분

🎯 이 모듈이 끝나면 — 터미널에서 코덱스가 실행되고, 옵시디언과 Claudian 플러그인이 설치되어 vault 안에서 Claude·Codex 채팅을 열 수 있는 상태가 됩니다.

💡 핵심 개념

Node.js는 도로, NPM은 그 도로를 달리는 택배 트럭입니다. 도로(Node.js)를 먼저 깔아야 트럭(NPM)이 코덱스 같은 도구를 배달할 수 있습니다.

CLI(Command Line Interface) = 마우스 대신 글자로 컴퓨터에 명령하는 창입니다. 검은 화면이 떠도 놀라지 않아도 됩니다.

NPM 명령 한 줄이면 도구가 설치됩니다. 이 과정에서 NPM으로 설치하는 것: 코덱스 CLI. 공식 인스톨러로 설치하는 것: Node.js, 옵시디언.

설치 순서가 중요합니다: Node.js → 코덱스 CLI → 옵시디언 → Claudian. 코덱스는 Node.js가 있어야 설치되고, 마지막으로 옵시디언 안에서 Claude·Codex를 쓰려면 Claudian 커뮤니티 플러그인을 설치합니다.

✅ 따라하기

Claudian 플러그인 설치 — 옵시디언에서 Claude·Codex 쓰기

Claudian은 Claude Code·Codex 같은 코딩 에이전트를 옵시디언 vault 안의 AI 협업자로 임베드하는 커뮤니티 플러그인입니다. 6단계에서 설치한 옵시디언에 이어서 진행합니다.

설치 완료 — 이제 왼쪽 리본의 Claudian 아이콘으로 채팅을 열어 Claude와 Codex를 탭별로 분리해 사용할 수 있습니다. 채팅 안에서 @로 vault 파일 멘션, /로 슬래시 커맨드, $로 Skills, #로 instruction 모드를 사용합니다.

출처 — Claudian: community.obsidian.md/plugins/realclaudian · GitHub: github.com/YishenTu/claudian

증상해결
😵 다운로드 페이지에 검은 코드 화면(Docker 등)만 보임그 박스는 다른 설치 방식 안내입니다. 무시하고 페이지를 아래로 내려 「Windows 설치 프로그램 (.msi)」 버튼을 클릭하세요.
😵 'node를 찾을 수 없습니다' 오류터미널을 완전히 닫고 새로 연 다음 다시 node -v. 설치 직후에는 기존 터미널이 새 프로그램을 모릅니다.
😵 npm install에서 권한 오류PowerShell을 관리자 권한으로 다시 실행했는지 확인하세요. (Windows에서 가장 흔한 원인)
😵 코덱스 로그인 실패모듈 A에서 메모해둔 ID/PW 사용. 브라우저에 다른 계정이 로그인돼 있으면 로그아웃 후 진행.
😵 claudian 검색 결과 중 뭘 설치할지 모름제작자가 Yishen Tu, 플러그인 ID가 realclaudian인 항목을 고르세요. 이름이 비슷한 다른 플러그인에 주의합니다.
😵 채팅창 모델 선택기에 Codex 모델이 안 보임Claudian 설정 → Codex 탭에서 "Enable Codex provider" 토글이 켜져 있는지 확인하세요(11단계).

🧠 스스로 확인 (질문을 클릭하면 답이 열립니다)

Q1. 다음 중 NPM으로 설치하지 않는 것은? (Node.js / 코덱스 CLI / 옵시디언)
Node.js와 옵시디언. Node.js는 NPM의 토대라서, 옵시디언은 NPM을 제공하지 않아서 각각 공식 인스톨러로 설치합니다.
Q2. Node.js를 코덱스보다 먼저 설치하는 이유는?
코덱스 CLI는 npm 명령으로 설치되는데, npm은 Node.js에 포함되어 있기 때문입니다. 도로가 있어야 택배가 옵니다.
