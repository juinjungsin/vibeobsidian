모듈 E. 자료 등록 — Ingest

웹 클리퍼로 자료를 수집하고, 확인 후 Ingest로 위키화한다 · ⏱ 50분

🎯 이 모듈이 끝나면 — 옵시디언 웹 클리퍼로 웹 자료를 수집하고, 클리핑된 자료를 확인한 뒤 /ingest로 위키 노트까지 만드는 수집→정제 순환을 수행할 수 있습니다. (내 컴퓨터 파일(PDF·PPT)의 변환 인제스트는 모듈 D에서 완료했습니다)

💡 핵심 개념

정유 공정의 첫 단계는 시추입니다. 웹 클리핑이 곧 시추 — 웹에서 원유(자료)를 퍼 올려 저장 탱크(10_Inbox)에 넣으면, 정제탑(/ingest)이 휘발유·경유(위키 노트 여러 개)로 분리합니다(1 대 다).

웹 클리퍼 = 웹페이지를 MD 노트로 오려오는 브라우저 확장. 본문만이 아니라 출처 URL·클리핑 날짜 같은 메타데이터를 함께 담아오기 때문에, 클리핑 자료는 인제스트 품질이 좋습니다.

모듈 D와의 차이 — D는 내 컴퓨터의 파일(PDF·PPT)을 변환해 등록했고, E는 웹 자료를 클리핑으로 수집해 등록합니다. 입구만 다르고 이후 공정(Raw 보관 → Wiki 분해)은 동일합니다.

클리핑은 정제의 시작 — 아무거나 모으지 않습니다(GIGO). 저장 버튼을 누르기 전 제목·본문을 확인하는 것이 1차 정제입니다.

핵심 규칙 유지 — 인박스에 들어가게 하는 것(클리핑)까지만 내 손으로, 이후의 모든 이동·분해는 클로디안에게 명령합니다.

✅ 따라하기

선택 — 클리핑 템플릿에 우리 프론트매터 규칙(15항목) 미리 적용하기

클리퍼 기본 템플릿의 속성(title·author·source·published·created·description·tags)은 우리 규칙 15항목과 다릅니다. 설정하지 않아도 됩니다 — /ingest가 부족한 항목을 채웁니다. 미리 맞춰두면 인제스트 때 AI가 묻는 질문이 줄어듭니다.

설정 위치: 클리퍼 설정 → 템플릿 → 기본(Default) 템플릿 → 속성(Properties). 기존 속성을 아래처럼 수정·추가하세요. {{ }}는 클리퍼가 페이지에서 자동으로 채우는 변수입니다.

title:        {{title}}
type:         raw
created:      {{date}}
origin:       external
confirmed:    false          (유형: 체크박스)
tags:         (빈칸)         (유형: 목록)
source_type:  웹페이지
source:       {{url}}
author:       {{author}}
published:    {{published}}
publisher:    {{site}}
summary:      {{description}}
key_question: (빈칸 — 검토할 때 직접 채움)
related:      (빈칸)         (유형: 목록)
status:       미검토

{{title}}=페이지 제목 · {{url}}=주소 · {{author}}=저자 · {{published}}=발행일 · {{site}}=사이트(발행처) · {{description}}=요약 · {{date}}=클리핑한 날짜. 변수가 비는 페이지도 있습니다 — 빈 항목은 /ingest 때 AI가 확인하거나 빈칸으로 유지됩니다(추측 금지 규칙).

증상해결
😵 클리퍼에서 저장했는데 옵시디언에 노트가 없음옵시디언이 꺼져 있으면 저장되지 않습니다. 옵시디언을 실행하고, 클리퍼 설정의 Vault 이름이 실제 Vault 이름과 정확히 같은지 확인하세요.
😵 클리핑이 10_Inbox가 아닌 곳에 저장됨클리퍼 설정에서 저장 위치(Note location)를 10_Inbox로 지정하세요. 이미 다른 곳에 저장된 노트는 클로디안에게 "방금 클리핑한 노트를 인박스로 옮기고 인제스트해줘"라고 명령하면 됩니다.
😵 본문이 비어 있거나 일부만 담김 (동적 페이지·이미지 글자)저장 전 미리보기에서 확인하세요. 비어 있으면 페이지에서 텍스트를 드래그 선택한 뒤 클리핑하거나, 페이지를 PDF로 저장해 모듈 D의 /convert 경로로 처리하세요.
😵 클리핑만 하고 아무 일도 안 일어남수집과 실행은 별개입니다. 클리핑 후 클로디안에 /ingest를 명령해야 위키가 만들어집니다.
😵 내가 직접 파일을 Raw 폴더로 옮김메타데이터가 빠진 채 들어가 검색·연결이 안 됩니다. 클로디안에게 "방금 옮긴 파일 다시 인제스트해줘"라고 명령하세요.

🧠 스스로 확인 (질문을 클릭하면 답이 열립니다)

Q1. 웹 클리퍼가 본문과 함께 담아오는 것은?
출처 URL·클리핑 날짜 같은 메타데이터. 이것이 프론트매터의 출처 항목(source·published) 재료가 되어, 손으로 복붙한 자료보다 인제스트 품질이 좋습니다.
Q2. 모듈 D의 인제스트와 이 모듈의 인제스트는 무엇이 다른가?
입구만 다릅니다. D는 내 컴퓨터 파일(PDF·PPT)을 변환해 넣었고, E는 웹 자료를 클리핑으로 넣습니다. 이후 공정(메타데이터 → Raw 보관 → 의미 단위 분할 → Wiki)은 동일합니다.
Q3. 클리핑 1건이 노트 여러 개가 되는 이유는?
의미 단위로 쪼개야 AI가 필요한 조각만 정확히 찾아 쓸 수 있기 때문입니다(정유 공정의 분별증류와 같은 원리 — 1 대 다).
