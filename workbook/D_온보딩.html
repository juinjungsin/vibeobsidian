모듈 D. 온보딩 — 스타터킷·스킬·변환

표준 폴더 구조를 자동 생성하고 변환 파이프라인을 갖춘다 · ⏱ 70~90분

🎯 이 모듈이 끝나면 — 표준 폴더 구조가 자동 생성되고, 내 파일을 MD로 변환할 수 있게 됩니다.

💡 핵심 개념

에베레스트를 오를 때 베이스캠프까지는 헬기로 갑니다. 스타터킷이 그 헬기입니다. 폴더 구조와 규칙을 직접 설계하는 것은 고급자의 일이고, 우리는 검증된 구조를 받아서 바로 등반(실습)을 시작합니다.

스타터킷 = 폴더 트리 + 폴더 간 규칙 + 자료 처리 방식을 미리 규정해둔 패키지. 받은 구조는 건드리지 않습니다.

폴더 이름 앞 숫자(10_, 20_…)는 순서 표시이자 길찾기용입니다. "50번 폴더 들어가세요"라고 안내받기 위함입니다.

스킬 = 클로디안에서 슬래시(/)로 부르는 자동화 명령. 긴 프롬프트를 복사해 붙이는 대신 /ingest처럼 한 단어로 실행합니다.

✅ 따라하기

선택 — OCR·변환 품질 높이기 (외부 도구, 설치하지 않아도 기본 변환은 동작합니다)

변환 스킬은 외부 도구가 없으면 AI가 직접 파일을 읽는 폴백으로 동작합니다. 스캔 PDF가 많거나 표가 복잡하다면 아래 도구를 쓰면 품질이 올라갑니다.

① 이미 설치했는데 "사용 불가"로 나오는 경우 — PATH 미등록이 원인입니다.

where tesseract
where soffice
python --version

안 나오는 도구는 시스템 환경 변수 PATH에 설치 경로를 추가한 뒤 터미널과 옵시디언을 재시작하세요. 예: C:\Program Files\Tesseract-OCR · C:\Program Files\LibreOffice\program. python이 "더미"로 나오면 Windows 설정 → 앱 실행 별칭에서 python 별칭을 끄세요.

② 도구가 없는 경우 (선택 설치)

LibreOffice(libreoffice.org — PPT·Excel·DOCX 변환), Tesseract OCR(UB-Mannheim 빌드 — 설치 시 Korean 언어 데이터 선택), Python(python.org — 설치 첫 화면에서 "Add python.exe to PATH" 체크).

설치 후(또는 도구가 있는데 인식이 안 될 때) 클로디안에 "변환 도구를 다시 탐지해줘"라고 요청하세요. 실제 진행 화면 예시:

Question
Tesseract 한글팩(kor) + pytesseract를 지금 설치할까요? (한글 OCR 보조 엔진용)
 1. 설치해줘 (권장) — kor.traineddata 내려받아 한글 OCR 가능하게 함
 2. 설치 안 함 — 한글 OCR은 AI 비전으로만 사용, 스킬은 지금도 정상 동작

이렇게 자연어로 요청하면 AI가 PATH 밖 설치 경로까지 찾아내고, 부족한 것(예: 한글팩)은 선택지로 물어봅니다. "설치해줘"를 고르면 설치까지 AI가 수행합니다 — 사용자가 직접 PATH를 만지는 것은 위 ①의 보조 수단입니다.

증상해결

😵 폴더가 자동 생성되지 않음스타터킷 내용물이 Vault 루트(.obsidian과 같은 층)에 있는지 확인하세요. 한 단계 위에 있거나 StarterKit 하위 폴더째 들어가 있으면 클로디안이 규칙 파일을 찾지 못합니다.

🧠 스스로 확인 (질문을 클릭하면 답이 열립니다)

Q1. 클로디안은 언제부터 설치할 수 있나?
Vault를 만들어 연 다음부터. 플러그인은 Vault 단위로 설치됩니다.
Q2. 받은 폴더 구조를 내 마음대로 바꾸면 안 되는 이유는?
규칙(메타데이터·자료 처리 방식)이 폴더 구조를 전제로 동작하기 때문입니다. 구조를 바꾸면 자동화가 깨지고, 직접 설계는 고급 과정의 일입니다.
