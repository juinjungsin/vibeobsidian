옵시디언 LLM Wiki·PKM·RAG 실습 Workbook

초보자가 따라하며 개념까지 익히는 모듈별 실습서 · A부터 O까지

🎯 이 과정의 목표 — 옵시디언을 AI가 읽고 쓰는 지식베이스(LLM Wiki)로 만들고, 내 자료로 프로젝트 산출물(PCD·MRD)을 뽑아내는 것. 핵심은 질문할수록 자산이 쌓이는 복리 구조입니다.

학습 흐름

입문: 준비(A) → 설치(B) → Vault(C) → 온보딩(D) → Ingest(E) → Query(F) → 개념(G) → 프로젝트(H)

후속: GraphRAG(I) → 세밀화(J) → 공유(K) → Index(L) → 결합(M) → 인사이트(N) → 종합(O) → 자동화(P) — 후속은 알파벳 순서(I→J→K→L→M→N→O→P)로 진행합니다.

진행 원칙: 일단 해본다 · 강사와 동일 환경 · 막히면 클로디안에게 자연어로 물어본다.

입문 과정

A. 사전 준비물
실습 환경 요건을 점검하고 계정을 준비한다 · ⏱ 15분
B. 설치
Node.js → 코덱스 CLI → 옵시디언 순서로 설치한다 · ⏱ 60~90분
C. Vault 생성
첫 보관소(Vault)를 만든다 · ⏱ 15분
D. 온보딩 — 스타터킷·스킬·변환
표준 폴더 구조를 자동 생성하고 변환 파이프라인을 갖춘다 · ⏱ 70~90분
E. 자료 등록 — Ingest
자료 1건을 의미 단위로 분할해 위키로 만든다 · ⏱ 50분
F. 질문과 답변 — Query & Compounding
질문하고, 가치 있는 답변을 다시 자산으로 쌓는다 · ⏱ 50분
G. 개념 정리 — LLM Wiki vs RAG·NotebookLM
방금 경험한 것의 원리를 구분한다 · ⏱ 40분
H. 산출물 문서 흐름 — 프로젝트 적용
위키 자산으로 PCD 초안을 만든다 · ⏱ 60분

후속 과정

I. GraphRAG — 관계가 답의 품질을 바꾼다
노트 간 '관계 이름'을 chunk로 활용한다 · ⏱ 40분
J. 세밀화 — 위키는 정제를 반복한다
노트가 정교해질수록 MD가 어떻게 달라지는가 · ⏱ 35분
K. Vault 공유 — 혼자에서 팀으로
공통 사용·Sync·오픈소스 분기를 이해한다 · ⏱ 30분
L. Index — 용어집과 색인
위키의 사전을 만든다 · ⏱ 30분
M. 그래프 Join — Vault 결합
Vault와 Vault를 연결해 시너지를 만든다 · ⏱ 35분
N. 그래프 인사이트 — View
연결의 공통분모에서 새 통찰을 꺼낸다 · ⏱ 30분
O. 프레임워크 + Obsidian — 종합
기존 산출물과 프레임워크를 결합해 최종 결과물을 만든다 · ⏱ 50분
P. 자동화 — 볼트가 스스로 자란다
스케줄로 리서치·Ingest·Query를 자동 순환, 사람은 승격만 한다 · ⏱ 50분

꼭 기억할 3가지 규칙

1. 인박스에 넣는 것까지만 내 손으로. 나머지는 전부 클로디안에게 명령합니다. 직접 폴더를 만지면 '쓰레기통'이 됩니다.

2. 정제된 자료만 넣습니다. 쓰레기를 넣으면 쓰레기가 나옵니다(GIGO). 오염은 미래의 답변까지 망칩니다.

3. 모르면 클로디안에게 물어봅니다. "지금 뭘 해야 해?", "니가 알아서 해"도 훌륭한 명령입니다.
