스타터킷 v1.0

📦 바이브 옵시디언 스타터킷

폴더 구조 + 폴더 간 규칙 + 자체 명령어(/ingest, /query)를 담은 시작 패키지

🎯 이것 하나로 — Vault에 복사하고 규칙을 적용하면, 합의된 파일 처리 흐름 Inbox → Raw → Wiki → Query → Output이 명령어 2개로 작동합니다. (모듈 D 온보딩에서 사용)

📁 폴더 구조

StarterKit/
├─ CLAUDE.md                 ← 클로드용 규칙(헌법)
├─ AGENTS.md                 ← 코덱스용 규칙(내용 동일)
├─ README.md                 ← 설치·온보딩 안내
├─ 00_Setup/commands/
│   ├─ ingest.md             ← /ingest 명령 원본 (온보딩 때 AI가 자동 설치)
│   └─ query.md              ← /query 명령 원본
├─ 10_Inbox/                 ← 자료 입장 (내 손이 닿는 유일한 폴더)
├─ 20_Raw/                   ← 메타데이터 장착 후 영구 보관·출처 추적
├─ 30_Wiki/
│   ├─ Concept/  Entity/  Guide/  MOC/
├─ 40_Query/                 ← 질문·답변 기록
└─ 50_Output/                ← 최종 산출물(PCD·MRD·보고서)

📜 폴더 간 규칙

폴더규칙
10_Inbox자료 입장. 사용자가 직접 파일을 넣는 유일한 폴더. 처리 후 원본은 삭제됩니다.
20_Raw메타데이터(프론트매터)가 장착된 MD 원본의 영구 보관소. 출처 추적 용도 — 수정·삭제 금지.
30_Wiki원본 1개가 의미 단위 N개로 분해된 지식 노트. 분류: Concept(개념) / Entity(실체) / Guide(방법) / MOC(목차·지도).
40_Query/query의 질문·답변 기록. 가치 있는 답변은 위키로 승격.
50_OutputWiki를 근거로 생성한 최종 산출물(PCD·MRD·보고서)이 저장. 지식의 집결지는 Wiki, 여기는 결과물 보관소.

⚡ 자체 명령어 2종

/ingest — Inbox 스캔 → MD 변환 → 프론트매터 장착(모르면 1·2·3 선택지로 질문) → Raw 영구 보관 → 의미 단위 N개 분해 → Concept/Entity/Guide 분류 + MOC 갱신 + 위키링크 연결(같은 개념은 누적, 충돌 시 사용자 판정) → Inbox 원본 삭제 → 결과 보고

/query 질문 — Wiki 안에서만 근거 탐색(외부 지식 금지, 근거 없으면 "없음"이라 답함) → 답변+근거 노트 목록 → 40_Query에 기록(origin: query-answer, confirmed: false) → 승격 지시 시 위키에 직접 생성/업데이트 + 컨펌 기록

명령어 원본은 00_Setup/commands/에 있습니다. 온보딩("규칙을 적용해줘") 때 AI가 .claude/commands/로 자동 복사해 클로드에서 정식 슬래시 명령으로 인식되고, 코덱스 연결 시에는 AGENTS.md 규칙으로 동작합니다. 사용자가 숨김 폴더를 만질 일은 없습니다.

🏷️ 프론트매터 표준 — 자료연구용 15항목 (노트의 신분증)

---
# ── 기본 (필수 · AI가 자동 기입) ──
title: 노트 제목
type: raw | concept | entity | guide | moc | query | output
created: YYYY-MM-DD            # Vault 등록일
origin: external | query-answer
confirmed: false
tags: []

# ── 출처 정보 (자료연구 · 자료에서 추출, 없으면 빈칸) ──
source_type: 논문 | 책 | 기사 | 웹페이지 | 영상 | 강의 | 회의록 | 보고서
source: 원본 파일명 또는 URL
author: 저자·발화자
published: YYYY-MM-DD          # 원자료 발행일 (연도만 알아도 기입)
publisher: 학술지·출판사·매체·기관

# ── 연구 메모 (선택) ──
summary: 한 줄 요약
key_question: 이 자료가 답하는 핵심 질문
related: []                    # 관련 노트 위키링크
status: 미검토 | 검토중 | 검토완료
---

총 15항목 = 기본 6(필수·자동) + 출처 5(자료에서 추출, 모르면 빈칸 — 추측 금지) + 연구 메모 4(선택, summary만 자동). 확인이 필요하면 AI가 1·2·3 선택지로 질문합니다.
