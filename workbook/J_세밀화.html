모듈 J. 세밀화 — 위키는 정제를 반복한다

노트가 정교해질수록 MD가 어떻게 달라지는가 · ⏱ 후속 1교시

🎯 이 모듈이 끝나면 — 노트 1개를 더 잘게·정확하게 다듬으며 위키가 살아있는 자산임을 체감합니다.

💡 핵심 개념

위키는 한 번 짓고 끝나는 건물이 아니라 계속 가지치기하는 정원입니다. 같은 노트도 정제를 거칠수록 답변 재료로서의 가치가 올라갑니다.

덩어리가 큰 노트는 검색 정확도를 떨어뜨립니다. 세밀화 = 더 작은 의미 단위로 재분할 + 프론트매터 보강.

나누는 방법은 둘입니다 — ① AI에게 "의미 단위로 쪼개줘"라고 맡기거나, ② 옵시디언 Note Composer 명령으로 내가 직접 떼어냅니다. 둘 다 목표는 같습니다: 한 노트 = 한 개념.

같은 개념이 자료마다 다르게 적혀 충돌하면, AI가 "뭐가 맞아요?"라고 묻고 전문가인 내가 판정해 정리합니다.

🧩 옵시디언 공식 명령어 — Note Composer

노트 분할·병합은 플러그인 설치 없이 옵시디언에 내장된 Note Composer 코어 플러그인으로 가능합니다(설정 → 코어 플러그인 → Note Composer, 기본 활성).

명령어 (명령 팔레트)동작
Note composer: Extract current selection
현재 선택 영역 추출		선택한 문단을 새 노트로 떼어내고, 원래 자리에는 그 노트로의 링크를 남깁니다. 큰 노트를 작은 단위로 쪼갤 때 쓰는 핵심 명령입니다.
Note composer: Merge current file with another file
다른 노트와 병합		너무 잘게 쪼갠 노트를 다시 합칩니다. 관련 링크는 자동으로 갱신됩니다.

실행: 명령 팔레트(Ctrl/Cmd + P)에서 명령 이름 입력 → Enter 대상 노트 끝에 추가 · Shift+Enter 앞에 추가 · Ctrl/Cmd+Enter 새 노트로 생성. 출처: help.obsidian.md/plugins/note-composer

⚠️ 여기서 막혀요

증상해결
😵 너무 잘게 쪼개져 산만해짐목적은 검색 정확도입니다. 한 노트 = 한 개념 수준이면 충분하고, 지나치면 합쳐달라고 하거나 Note composer: Merge로 되돌리면 됩니다.
😵 Extract·Merge 명령이 팔레트에 안 보임설정 → 코어 플러그인에서 Note Composer가 켜져 있는지 확인하세요. 꺼져 있으면 두 명령이 나타나지 않습니다.

🧠 스스로 확인 (질문을 클릭하면 답이 열립니다)

Q1. 노트가 클수록 생기는 문제는?
질문과 관련 없는 내용까지 함께 딸려 와 답변 정확도가 떨어집니다.
Q2. 노트를 손으로 더 작게 나누는 옵시디언 공식 명령은?
Note Composer 코어 플러그인의 Extract current selection — 선택한 부분을 새 노트로 떼어내고 원래 자리에 링크를 남깁니다. 반대로 합칠 땐 Merge current file with another file를 씁니다.
